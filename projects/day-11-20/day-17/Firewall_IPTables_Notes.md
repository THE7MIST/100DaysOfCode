# Firewall and IPTables Notes

## Firewall

A firewall is a hardware or software security device that monitors and controls incoming and outgoing network traffic based on predefined security rules.

It separates a trusted network/environment from an untrusted network/environment and allows or denies traffic according to configured policies.

---

# Types of Firewall

## 1. Network Firewall

A network firewall protects an entire network.

It is placed between trusted and untrusted networks and handles both incoming and outgoing traffic.

### Examples of Untrusted Networks

* Internet
* Branch Office Network
* Customer Network
* Vendor Network

### Functions

* Monitors network traffic
* Allows authorized traffic
* Blocks unauthorized traffic
* Protects multiple systems simultaneously

---

## 2. Host Firewall

A host firewall is installed on an individual computer or server.

It protects the host by filtering traffic entering or leaving that particular system.

### Functions

* Controls inbound traffic
* Controls outbound traffic
* Protects individual computers
* Adds an extra security layer

### Examples

* Windows Defender Firewall
* Linux iptables
* UFW

---

# First Generation Firewall (Packet Filtering Firewall)

Packet Filtering Firewall works up to the Transport Layer (Layer 4) of the OSI Model.

### Filters Based On

* Source IP Address
* Destination IP Address
* Source Port Number
* Destination Port Number
* Protocol Type

### Protocol Examples

* TCP
* UDP
* ICMP

### Advantages

* Fast processing
* Low resource consumption

### Disadvantages

* Cannot inspect packet contents
* Limited security

# Built-in Chains

There are 5 built-in chains in iptables.

## PREROUTING

Packet enters the system before routing decisions.

## INPUT

Packet destined for the local machine.

Examples:

* SSH to server
* Ping server

## FORWARD

Packet passing through the firewall/router.

Example:

```text
Client → Firewall → Internet
```

## OUTPUT

Packet generated by the local machine.

Examples:

* Ping from server
* curl command

## POSTROUTING

Packet processing after routing decision and before leaving the interface.

---

# Packet Flow

```text
Incoming Packet
      ↓
PREROUTING
      ↓
Routing Decision
      ↓
 ┌───────────┬───────────┐
 ↓           ↓
INPUT      FORWARD
 ↓           ↓
Local     POSTROUTING
Host         ↓
         Destination
```

For locally generated traffic:

```text
OUTPUT
   ↓
POSTROUTING
   ↓
Network
```

# IPTables Tables

There are 5 major tables.

## 1. Filter Table (Default)

Purpose:

* Allow Traffic
* Deny Traffic

Chains:

* INPUT
* FORWARD
* OUTPUT

Default table used by iptables.

---

## 2. NAT Table

Purpose:

* Network Address Translation

Chains:

* PREROUTING
* POSTROUTING
* OUTPUT

Examples:

* SNAT
* DNAT
* Masquerading

---

## 3. RAW Table

Purpose:

Used before connection tracking.

Allows packets to bypass connection tracking.

Chains:

* PREROUTING
* OUTPUT

---

## 4. Mangle Table

Purpose:

Modify packet parameters.

Examples:

* TTL
* QoS
* TOS
* DSCP

Chains:

* PREROUTING
* INPUT
* FORWARD
* OUTPUT
* POSTROUTING

---

## 5. Security Table

Purpose:

Used primarily by SELinux for Mandatory Access Control (MAC).

Chains:

* INPUT
* OUTPUT
* FORWARD

---

# Common IPTables Commands

## Specify Table

```bash
iptables -t <table-name>
```

Example:

```bash
iptables -t nat
```

---

## Append Rule

```bash
iptables -A <chain-name> <rule>
```

Example:

```bash
iptables -A INPUT -s 192.168.1.10 -j DROP
```

---

## Replace Rule

```bash
iptables -R <chain-name> <rule-number> <rule>
```

Example:

```bash
iptables -R INPUT 3 -s 192.168.1.10 -j DROP
```

---

## Insert Rule

```bash
iptables -I <chain-name> <position> <rule>
```

Example:

```bash
iptables -I INPUT 2 -s 192.168.1.10 -j DROP
```

---

## Delete Rule

```bash
iptables -D <chain-name> <rule-number>
```

Example:

```bash
iptables -D INPUT 1
```

---

# Stateful Firewall

Stateful firewalls maintain a state table of active connections.

Example:

```text
192.168.10.10:3100
        ↓
20.20.20.20:80
```

State Transition:

```text
NEW → ESTABLISHED
```

---

# Connection States

## NEW

New connection request.

Example:

SSH connection.

## ESTABLISHED

Existing active connection.

## RELATED

Related to an existing connection.

Example:

FTP data connection.

## INVALID

Malformed or unknown packet state.

Usually dropped.

---

# Useful Stateful Rules

Allow existing connections:

```bash
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
```

Drop invalid packets:

```bash
iptables -A INPUT -m state --state INVALID -j DROP
```
